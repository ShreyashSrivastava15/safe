# 🛡️ S.A.F.E. Project: Master Interview Guide

This guide contains everything you need to know to ace an interview about the **S.A.F.E. (Scam Analysis and Fraud Elimination)** project.

---

## 1. Project High-Level Overview
**S.A.F.E.** is a multi-modal, real-time fraud detection platform designed to protect users from modern digital threats across five critical vectors: **Email Phishing**, **Malicious URLs**, **Financial Transaction Fraud**, **E-Commerce Scams**, and **Social Engineering**.

- **Goal**: To provide a unified "Forensic Verdict" with human-readable explanations using advanced AI.
- **Unique Selling Point (USP)**: Unlike simple blacklists, S.A.F.E. uses a **Weighted Fusion Engine** that combines Natural Language Processing (NLP), Behavioral Anomaly Detection, and Technical Heuristics.

## 3. Core Features & "Wow" Factors
### 🛰️ Real-time Gmail Monitoring
- Integrated with Google Cloud Console.
- Uses **Webhooks** to listen for new emails.
- Pushes live notifications to the dashboard using **SSE** (Server-Sent Events), so the user doesn't have to refresh.

### 📱 SMS Neural Bridge
- A custom-built simulation for mobile fraud.
- Allows users to link their device (simulated) and sync suspicious text messages for analysis.
- Includes custom mock data for specific numbers like `+919769039702` (SBI/ICICI scams).

### 📊 Forensic Analysis Page
- **Risk Score Gauge**: Visual 0-100 gauge showing the threat level.
- **Intelligence Summary**: A human-readable "explanation" generated by the AI.
- **Evidence Chain**: A breakdown of specific findings (e.g., "Urgency detected," "Suspicious TLD").

### 📄 Professional Exporting
- Integrated **jsPDF** and **html2canvas** to generate professional forensic reports as PDF files.

## 4. The AI Engine Deep-Dive (Technical)
The project uses a **Multi-Engine parallel architecture**:

1.  **NLP Engine (Communication)**:
    *   Uses a **BERT-tiny** transformer model (fine-tuned for spam/phishing).
    *   Analyzes intent: Urgency, Financial pressure, or Predatory language.
2.  **Transaction Engine (Financial)**:
    *   Uses **Isolation Forest** (Unsupervised Learning) to detect behavioral anomalies.
    *   Features: Amount, Velocity, Geo-shift, and Device Fingerprint.
3.  **URL Engine (Technical)**:
    *   Analyzes **Entropy** (randomness of domain), **TLD Reputation** (.xyz, .shop), and **Deception Patterns** (like the `@` symbol trick).

## 5. System Architecture
> [!IMPORTANT]
> **Orchestration Pattern**: The Node.js gateway acts as the **Orchestrator**. When a request comes in, it splits the data and calls the Python FastAPI engines in parallel. It then aggregates the scores using weights defined in `riskConfig.ts`.

## 7. Common Interview Questions
**Q: Why use FastAPI for AI instead of just putting it in Express?**
*A: Separation of Concerns. Python has the best ML ecosystem (Torch, Scikit-learn). FastAPI is asynchronous and highly efficient for high-latency AI inference tasks, while Node is better at I/O and user management.*

**Q: How do you handle false positives?**
*A: We have a **Reputation Layer**. For example, if an email is from a trusted domain (`google.com`) and passes **DKIM/SPF verification**, the risk score is automatically reduced, even if the language seems urgent.*

**Q: What would you add if you had more time?**
*A: I would implement **Multi-lingual support** for non-English fraud and integrate a **Browser Extension** for real-time protection while surfing.*
